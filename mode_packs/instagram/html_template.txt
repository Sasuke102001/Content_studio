You are designing a premium Instagram carousel for Polynovea — a behavioral intelligence and decision infrastructure company.

The visual system must communicate:
- institutional intelligence
- operational maturity
- structural precision
- premium restraint

NOT:
- startup energy
- music brand aesthetics
- generic AI visuals
- cinematic overdesign
- decorative clutter

## FORMAT
This is an Instagram carousel.
Each slide:
- Width: 420px
- Height: 525px (4:5 aspect ratio)
- Total slides: 6
- All slides rendered as a single scrollable HTML page (stacked vertically)
- Each slide uses class="slide"

Navigation:
- Progress dots at the bottom of each slide showing current position
- Slides should feel like a contained, intentional sequence

## COLOR SYSTEM
Use the PRIMARY Polynovea brand palette (dark system).
Background layers:
- Primary background: #0A0A0A
- Elevated surface: #121212
- Card surface: #18181B
- Border: #27272A
Typography:
- Primary text: #F5F5F5
- Secondary text: #A1A1AA
- Muted text: #71717A
Accent layers:
- Authority accent (gold): #E6D3A3
- Muted gold: #9A8F6A
- Depth accent (violet): #7C3AED

Rules:
- Dark backgrounds are non-negotiable for this carousel — do NOT use the light/warm presentation palette
- Gold is the primary accent for headlines and key emphasis
- Violet is used sparingly: flow arrows, intelligence markers, slide 3 only
- No pure white backgrounds
- No startup color gradients

## TYPOGRAPHY
Fonts:
- Clash Display — all headlines and eyebrow labels
- Inter — all body copy

Load fonts via:
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600&display=swap" rel="stylesheet">
<link href="https://api.fontshare.com/v2/css?f[]=clash-display@400,500,600,700&display=swap" rel="stylesheet">

Scale hierarchy:
- Eyebrow label: Clash Display, 10px, letter-spacing: 0.15em, uppercase, muted gold (#9A8F6A)
- Headline (H1): Clash Display, 26–32px, weight 600, primary text (#F5F5F5)
- Subheadline: Clash Display, 18–22px, weight 500
- Body copy: Inter, 13–15px, weight 400, secondary text (#A1A1AA), line-height 1.7
- Company handle: Inter, 11px, muted text (#71717A)

Rules:
- Headlines should feel dominant and intentional
- Body copy must be readable at phone size — no text below 12px
- No walls of text — 2–3 short lines per body block maximum
- Tight letter spacing on headlines (-0.02em)
- Generous line height on body (1.7+)

## LAYOUT SYSTEM
Each slide has:
- 36px padding on all sides
- A top zone: eyebrow label (left-aligned)
- A mid zone: headline (left-aligned, strong visual weight)
- A lower zone: body copy and any supporting visual structure
- A bottom strip: progress dots (centered, 8px from bottom)

Slide variation rules — no two consecutive slides should feel structurally identical. Rotate through:
1. Full-width centered text with large headline
2. Left-aligned editorial with right side structural element
3. Full headline dominating upper 60%, minimal body below
4. Split zone: headline top-left, body bottom-right
5. Structural layout with a flow diagram or minimal framework visual
6. Strong close: headline dominant, handle and progress indicator

## PROGRESS DOTS
Each slide shows its position in the sequence via dots at the bottom center.
- Dot size: 5px circle
- Active dot: #E6D3A3 (gold), 5px
- Inactive dots: #27272A (border color), 5px
- Gap between dots: 6px
- Position: 12px from bottom edge, centered

## VISUAL DETAIL SYSTEM
Use these sparingly to add depth without clutter:
- Horizontal rules: 1px, #27272A, full width — for section separation
- Minimal card blocks: #18181B background, 1px border #27272A, 6px border radius
- Label tags: small rounded pill in #18181B with 1px gold border, 10px Clash Display text
- Flow arrows: only on slide 4, in violet (#7C3AED), minimal and clean

Avoid:
- Stock-style imagery
- Icon overload
- Gradient backgrounds
- Drop shadows that feel soft or corporate
- Decorative elements without structural purpose

## VISUAL COMPONENT LIBRARY
When a slide's lower zone needs a "supporting visual structure," use ONE of these
content-ful patterns — adapt numbers/labels/text to the slide's argument. Every
pattern renders with real content. Never substitute an empty box or placeholder card.

1. METRIC CARD — large numeral + small label:
<div style="background:#18181B;border:1px solid #27272A;border-radius:6px;padding:16px 18px;display:inline-block;">
  <div style="font-family:'Clash Display';font-size:28px;font-weight:600;color:#E6D3A3;line-height:1;">3.2x</div>
  <div style="font-family:Inter;font-size:10px;color:#A1A1AA;margin-top:6px;letter-spacing:0.05em;">Faster decision cycles</div>
</div>

2. SIGNAL BARS — row of 5-8 bars with explicit varying heights:
<div style="display:flex;align-items:flex-end;gap:6px;height:52px;">
  <div style="width:7px;height:24px;background:#7C3AED;opacity:0.5;border-radius:2px;"></div>
  <div style="width:7px;height:38px;background:#7C3AED;opacity:0.7;border-radius:2px;"></div>
  <div style="width:7px;height:52px;background:#E6D3A3;opacity:0.9;border-radius:2px;"></div>
  <div style="width:7px;height:30px;background:#7C3AED;opacity:0.6;border-radius:2px;"></div>
</div>
(Always set distinct height values per bar — never identical or zero-height bars.)

3. FLOW / FRAMEWORK ROW — 2-4 labeled nodes connected by arrows (generalizes slide 4's Pattern → Intelligence → Decision):
<div style="display:flex;align-items:center;gap:8px;">
  <div style="background:#18181B;border:1px solid #27272A;border-radius:6px;padding:8px 12px;font-family:Inter;font-size:10px;color:#E6D3A3;">Pattern</div>
  <span style="color:#7C3AED;font-size:13px;">&#8594;</span>
  <div style="background:#18181B;border:1px solid #27272A;border-radius:6px;padding:8px 12px;font-family:Inter;font-size:10px;color:#E6D3A3;">Decision</div>
</div>

4. LABEL/STATUS STRIP — pill tag(s) with a category marker (generalizes slide 5's "Live Environments" tag):
<div style="display:inline-block;background:#18181B;border:1px solid #9A8F6A;border-radius:20px;padding:6px 14px;font-family:'Clash Display';font-size:10px;color:#E6D3A3;letter-spacing:0.1em;text-transform:uppercase;">Live Environments</div>

HARD RULE — NO EMPTY CONTAINERS:
Never output a <div> or container that has no text content, no <svg>, and no
background-image/gradient/pattern. Every element in the lower zone must either
(a) contain real text or numbers, or (b) be one of the Visual Component Library
patterns above, fully populated with content. Thin 1px horizontal rule lines and
progress dots are exempt — those are intentional structural elements, not
placeholders. If a slide's content doesn't naturally fill the canvas, prefer
variation rule 1 or 3 (headline-dominant) over adding a decorative empty space-filler.

## MOTION
No animations required for this output.
Generate a static HTML file with all 6 slides rendered vertically in sequence.
Each slide should be visually complete as a standalone unit and work as a screenshot for Instagram posting.
