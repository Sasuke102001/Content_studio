Implementation Notes

Naming Convention: All tokens use role-based naming (not value-based). This allows palette swaps without breaking implementation. Format: [category]-[role] where category is bg, text, border, accent, signal.
Dark System (Primary): Use for website, social media, ecosystem visuals, dashboards, and all digital interfaces. The deep black foundation reduces visual noise and emphasizes premium positioning.
Light System (Secondary): Use exclusively for presentations, PDFs, client documents, and decision-making environments. The warm off-white creates readability-first hierarchy for printed and projection contexts.
Functional Signals (Light Palette Only): The three signal colors (positive, warning, risk) are defined only for the light palette. For dark contexts, use accent-authority (gold) and accent-depth (violet) for emphasis. Do not use green/amber/red in dark interfaces.
Shared Accents: accent-authority (#E6D3A3, gold) and accent-intelligence (#7C3AED, violet) are identical across both systems, creating visual continuity. Use gold sparingly for authority signals; reserve violet for intelligence markers, graph highlights, and system callouts.
Interactive States (Dark System): accent-authority-hover (#F0DFB8) provides feedback on gold-colored interactive elements. border-active (#7C3AED) marks focus states and active elements using violet — the system signal color. Blue has no role in the Polynovea palette.
